# AirWise Founder Execution Sheet

Last updated: 2026-04-15

## Purpose

This sheet reflects the current company execution reality: AirWise already has a working local MVP, so founder effort should shift from “stand up the repo” to “tighten the product truth, deepen filing workflows, and make pilot execution smoother.”

## Current phase

AirWise is in a transition between:

- local MVP assembly
- pilot workflow hardening

The product already spans compliance, filing, documents, monitoring, recommendations, and supervised commands. The next founder focus is improving clarity, trust, and operational readiness.

## Founder split

### Technical founder

Owns:

- product truth in docs and UI
- code quality and architecture
- reporting/f filing workflow depth
- monitoring/runtime reliability
- pilot demo environment quality

### Domain / GTM founder

Owns:

- pilot intake quality
- building readiness validation
- customer-facing workflow language
- filing process reality checks
- turning product surfaces into a credible pilot narrative

## Current top priorities

### Product / engineering

- keep docs aligned with what exists in the product
- improve no-sensor and BAS-only onboarding clarity
- reduce ambiguity inside the filing workspace
- strengthen command/runtime observability

### Pilot / GTM

- keep intake expectations realistic for data-light buildings
- use overview + filing + compliance as the opening wedge
- position monitoring as a second-step expansion when BAS access exists
- limit supervised control promises to whitelisted pilot cases

## Weekly operating cadence

Every week should produce:

- one demoable product improvement
- one pilot-facing workflow clarification
- one risk retired from either filing or runtime operations

## Founder review questions

- Is the repo documentation still describing the actual shipped product?
- Can a no-sensor building still get a meaningful outcome this week?
- Are filing workflows more complete than they were last week?
- Can the team explain the monitoring and command boundaries without hand-waving?
- Are pilot claims narrower than the current product, never broader?
