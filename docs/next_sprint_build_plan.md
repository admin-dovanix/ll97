# AirWise Next Sprint Build Plan

Last updated: 2026-04-15

## Sprint objective

Turn the newly added BAS profile and filing foundations into a clearer onboarding path for no-sensor and BAS-only buildings.

## Current baseline

Already implemented:

- BAS/system profile fields on the building overview
- filing workspace and reporting-cycle shell
- compliance, documents, and import review surfaces
- gateway-backed monitoring and supervised command scaffolding

## Sprint focus

This sprint should improve product clarity, not invent a new platform.

### In scope

- clearer readiness language on top of current BAS profile data
- stronger overview-page guidance for buildings without telemetry
- better filing workspace expectations for manual-first package assembly
- better portfolio understanding of mixed building maturity

### Out of scope

- production auth and SSO
- protocol expansion beyond the BACnet-first path
- autonomous optimization
- broad supervised-control expansion

## Desired outcome

By the end of the sprint, a no-sensor or BAS-only building should feel intentionally supported, not like an incomplete monitoring account.
