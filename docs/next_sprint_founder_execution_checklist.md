# AirWise Next Sprint Founder Execution Checklist

Last updated: 2026-04-15

## Sprint purpose

Make the current product easier to use for buildings that start without telemetry or without an active gateway runtime.

## CEO / product checklist

- define the user-facing readiness language that sits on top of existing BAS profile fields
- make sure no-sensor buildings have an intentional path through overview, compliance, documents, and filing
- review whether the filing workspace wording matches actual 2026 preparation use

## Engineering checklist

- keep docs aligned with the implemented routes and data model
- tighten overview-page BAS/readiness UX
- reduce ambiguity in filing workspace statuses and blockers
- preserve current monitoring and command flows while improving onboarding clarity

## GTM / pilot checklist

- validate the new readiness framing with real pilot conversations
- keep monitoring promises conditional on BAS/runtime access
- use filing preparation, not telemetry, as the opening wedge where appropriate
